# Requirement Index

## Phase 4B — Client self-service completion

- **REQ-004**: Authenticated client account shell must expose appointments, manage codes, and logout.
- **REQ-005**: Authenticated appointments list must segment upcoming vs past and preserve appointment aggregate identity.
- **REQ-006**: Authenticated surfaces must provide edit info, reschedule, and cancel entrypoints linked to appointment identity.
- **REQ-007**: Manage codes must provide non-empty and empty-state handling.
- **REQ-008**: Invalid or missing client auth token must route to explicit not-authorized self-service state.
- **REQ-029**: Mobile-responsive parity must reuse the same self-service business logic as desktop.
