# Requirement Index

## Phase 10A — Reports Foundation (in scope)

- **REQ-RPT-001**: Reports shell foundation must exist as a first-class admin surface.
- **REQ-RPT-002**: Support report tabs for Appointments, Revenue, Users, Intake Forms, and Tips.
- **REQ-RPT-003**: Shared report filters must include date range and calendar filtering.
- **REQ-RPT-004**: Reports must render valid empty states for sparse datasets.
- **REQ-RPT-005**: Intake Forms report must remain blank until explicit question selection and Show action.
- **REQ-RPT-006**: Reporting access must be server-aware gated by role, plan/trial, dependency state, and verification state.
- **REQ-RPT-007**: Appointment remains the central aggregate boundary for report-linked data modeling.
- **REQ-RPT-008**: Reporting surfaces remain separated from Manage Users and account billing shell surfaces.

## Out of scope for Phase 10A

- Manage Users implementation.
- Host/account billing shell implementation.
- Full Import/Export workflow implementation beyond report foundations.
